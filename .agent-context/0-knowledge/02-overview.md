# Lyra - Overview & Key Principles

## Overview

Lyra is a project planning tool designed to handle maintenance of plans for single or multiple plans. It is flexible and configurable.

## Key Components

- **Landing page**
- **Home page**: Choice between hierarchical map of user's accessible projects and alphabetical list of user's accessible projects
- **Plan maintenance**: Day-to-day running of the plan using gantt or task rows
- **Change control**: Audit log and change management — user sets threshold
- **Dependency management**: Separate workspace to decide on task relationships before publishing to the plan, all decisions being easily reversible
- **Reporting**: Flexible, configurable, designed for the boardroom. Canned reports available for weekly/monthly status reporting
- **Notifications**: Configurable notifications can be emailed to task owners (e.g. past due)
- **AI Plan Generation**: First draft of plan compiled based on existing project materials (e.g. SOW)
- **APIs**: Works with Teams; data can be uploaded to Planview, Clarity etc.
- **Product management**: Kanban view — tag which plan tasks have a kanban associated — status maintained across plan and gantt
- **Calendar view** of tasks
- **R&I register**: Automatically generate risk/issue for red planning items. Status commentary needed once, at detailed task level, and AI can do the aggregation for workstream/project/programme/portfolio. This is a time-consuming task for PMs and workstream leads.

## Key Principles

- **Flexibility** — configurable by user
- **Exec reporting** has to look best in class and be usable in the boardroom
- **Change control** — real workflow
- **Dependencies** have to be useful
- **AI plan generation**
